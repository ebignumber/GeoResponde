# How You Can Help

GeoResponde is more than a software project.

It is an open platform designed to improve access to scientific and humanitarian information during disasters.

Whether you're a developer, scientist, GIS professional, humanitarian organization, designer, or volunteer, there are many ways you can contribute.

---

## Branch Naming

Please do not commit directly to `main`.

Create a dedicated branch for every contribution.

Use the following naming convention:

- `feature/...` — New features
- `provider/...` — New humanitarian provider integrations
- `layer/...` — New scientific or geospatial layers
- `sdk/...` — Provider SDK improvements
- `fix/...` — Bug fixes
- `docs/...` — Documentation changes

Examples:

provider/terremotovenezuela
provider/cruz-roja
layer/geofon
feature/mobile-responsive
fix/search-scroll
docs/readme

# Our Philosophy

GeoResponde does not seek to replace existing organizations.

Instead, we aim to improve interoperability between them.

Our mission is simple:

- Connect information.
- Respect data ownership.
- Reduce duplicated effort.
- Improve situational awareness.
- Help people find trusted information faster.
